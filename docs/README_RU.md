## 📖 Описание

Разблокировщик DLC для **Europa Universalis IV** на **macOS**. Использует
[CreamAPI v5.3.0.0](https://cs.rin.ru/forum/viewtopic.php?f=29&t=70576) для
разблокировки владения DLC в игре, легально купленной в Steam.

### Как это работает

Инструмент устанавливает `libsteam_api.dylib` от CreamAPI (прокси перед
настоящей библиотекой Steam) в `eu4.app/Contents/Frameworks/`, сохраняет
оригинал как `libsteam_api_o.dylib` и пишет `cream_api.ini`. При запуске игра
спрашивает Steam, какими DLC вы владеете; CreamAPI отвечает «да» по каждому
разблокированному DLC, а всё остальное проксирует в настоящий Steam — поэтому
**мультиплеер продолжает работать**.

При установке инструмент **читает метаданные `dlc/*.dlc` самой игры** и пишет
явный список `[dlc]` (`unlockall = false`). Это сделано намеренно: у EU4 130+
паков контента, а режим `unlockall = true` опирается на *рантайм*-список DLC от
Steam, который для игр с большим числом DLC приходит **обрезанным** — тогда
разблокируется лишь часть. Список, собранный из вашего же контента, **полный** и
**работает офлайн** после первого запуска. Он пересобирается при каждой
установке, так что после добавления нового контента достаточно переустановить.
Если метаданных DLC не найдено — откат к `unlockall = true`.

### Возможности

- ✅ **Мультиплеер поддерживается** (CreamAPI проксирует настоящий Steam)
- ✅ Полная разблокировка DLC — явный список из вашего же контента `dlc/` (без обрезки, работает офлайн), пересобирается при каждой установке
- ✅ Автоопределение установки EU4 (внутренний диск **и** внешние накопители)
- ✅ Безопасный бэкап оригинала; чистое обратимое удаление
- ✅ Команда `status` без изменений — проверка того, что установлено
- ✅ Проверяемая по контрольной сумме **универсальная** сборка (Intel **и** Apple Silicon)

## 📋 Требования

### Поддерживаемые платформы

| Платформа    | Архитектура           | Статус                  |
|--------------|-----------------------|-------------------------|
| macOS 10.13+ | Intel (x86_64)        | ✅ Нативно              |
| macOS 11+    | Apple Silicon (M1–M4) | ✅ Нативно (arm64)      |

Поставляемая `libsteam_api.dylib` — **универсальный** (fat) бинарник с обоими
слайсами `x86_64` и `arm64`, поэтому **Rosetta 2 на Apple Silicon не нужна**.

### Необходимое

- macOS 10.13 (High Sierra) или новее
- Europa Universalis IV, легально купленная в Steam
- **Файлы контента DLC** (должны уже присутствовать — инструмент их не качает)
- Желательны Xcode Command Line Tools (`xcode-select --install`) для `codesign`/`xattr`

## 🚀 Установка

**Коротко** — три команды, потом запустить игру:

```bash
git clone https://github.com/KyLaEga/EU4-DLC-Unlocker-macOS.git
cd EU4-DLC-Unlocker-macOS
./install_dlc_unlocker.sh
```

### По шагам

1. **Сначала убедитесь, что контент DLC на месте.** Папка `dlc/` игры уже должна
   содержать паки DLC (инструмент разблокирует владение, но **не качает**
   контент). Если папка пуста — см.
   [Где взять файлы контента DLC?](#где-взять-файлы-контента-dlc) ниже.

2. **Склонируйте или скачайте** репозиторий:
   ```bash
   git clone https://github.com/KyLaEga/EU4-DLC-Unlocker-macOS.git
   cd EU4-DLC-Unlocker-macOS
   ```

3. **Запустите установщик:**
   ```bash
   ./install_dlc_unlocker.sh
   ```
   Это тонкая обёртка над `./bin/unlocker install eu4`. Инструмент проверяет
   поставляемую библиотеку, автоопределяет установку EU4 (внутренний диск **и**
   внешние накопители), сохраняет оригинал как `libsteam_api_o.dylib`, ставит
   CreamAPI, собирает явный список DLC из вашего `dlc/` и ad-hoc переподписывает
   результат для Gatekeeper.

   Если игра в нестандартном месте — укажите путь напрямую:
   ```bash
   ./bin/unlocker install eu4 --path "/Volumes/My Drive/SteamLibrary/steamapps/common/Europa Universalis IV"
   ```

4. **Запустите EU4 через Steam.** С явным списком `[dlc]` это работает и офлайн.
   **Важно:** если игра или Paradox Launcher уже были открыты — полностью
   **закройте их (`⌘Q`)** и запустите заново: CreamAPI читает `cream_api.ini`
   только один раз при старте процесса.

### Проверка

Посмотреть состояние, ничего не меняя:
```bash
./bin/unlocker status eu4
```
Должно быть так:
```
[+] Patch state:     INSTALLED (CreamAPI active)
[+] Original backup: PRESENT (...libsteam_api_o.dylib)
[+] cream_api.ini:   PRESENT
[*]   unlockall = false; explicit [dlc] list (N DLC)
```
Затем откройте вкладку **DLC** в Paradox Launcher — все DLC, чей контент есть,
должны стать купленными (без кнопок «Buy» и без жёлтых предупреждений).

## 🛠️ Использование (CLI `unlocker`)

Единая точка входа — `bin/unlocker`; два скрипта `*_dlc_unlocker.sh` оставлены
как тонкие обёртки для обратной совместимости.

```text
./bin/unlocker install   [eu4]   # бэкап оригинала + установка CreamAPI
./bin/unlocker uninstall [eu4]   # восстановить оригинальную библиотеку
./bin/unlocker status    [eu4]   # без изменений: патч / бэкап / целостность
./bin/unlocker --help
./bin/unlocker --version

Опции (после команды):
  --yes          Без вопросов; считать ответ «да».
  --path DIR     Использовать эту папку игры вместо автоопределения.
```

Проверить состояние, ничего не меняя:
```bash
./bin/unlocker status eu4
```

### Что делает `install` под капотом

1. **Проверка целостности** — не запустится, если SHA256 поставляемой
   `libsteam_api.dylib` не совпадает с зафиксированным (защита от подмены).
2. **Поиск игры** — разворачивает пути из `games/eu4.conf` (с пробелами и
   внешними дисками `/Volumes/*`); либо берёт `--path`.
3. **Бэкап** оригинала как `libsteam_api_o.dylib` (не перезаписывает хороший).
4. **Установка CreamAPI** — копирует прокси-dylib и перепроверяет хэш копии
   перед переподписью.
5. **Сборка `cream_api.ini`** — сканирует `dlc/*/*.dlc`, собирает `steam_id`,
   убирает дубли и пишет явный список `[dlc]` с `unlockall = false`.
6. **Защита под macOS** — снимает карантин (`xattr`) и ad-hoc переподписывает
   dylib (`codesign -s -`), чтобы Gatekeeper загрузил её в процессе игры.

`uninstall` отменяет шаги 3–4 (вернуть бэкап, удалить `cream_api.ini`,
переподписать). `status` показывает всё это и ничего не меняет.

## 🗑️ Удаление

```bash
./uninstall_dlc_unlocker.sh
# или: ./bin/unlocker uninstall eu4
```

Восстанавливает `libsteam_api.dylib` из бэкапа, удаляет `cream_api.ini`,
переподписывает оригинал и убирает устаревшие артефакты.

> Запуск в Steam **«Проверить целостность файлов игры»** молча восстанавливает
> оригинальную библиотеку и снимает патч — это нормально. Запустите установщик
> снова, чтобы вернуть патч, или `status`, чтобы проверить состояние.

## ❓ FAQ

### У DLC значки-предупреждения в лаунчере
Отсутствуют **файлы контента** DLC. Инструмент меняет только статус владения —
сам контент должен лежать в папке `dlc/` игры.

### Где взять файлы контента DLC?
Инструмент **не распространяет и не качает** DLC — контент вы приносите сами.
Контент DLC от Paradox кросс-платформенный, поэтому обычный путь — **скопировать
папку `dlc/` с машины, где он у вас уже есть** (например, ваша копия на
Windows/Linux) в установку на Mac:
```
.../steamapps/common/Europa Universalis IV/dlc/
```
Каждый DLC лежит в своей подпапке (`dlc/dlcNNN_name/`) с файлом метаданных
`.dlc`, архивом `.zip` и превью. После копирования запустите (или перезапустите)
установщик, чтобы список `[dlc]` пересобрался под новый контент.

**Контент пришёл многотомным архивом (`*.zip.001`, `*.zip.002`, …)?**
Это **один** архив, нарезанный на куски, а не отдельные зипы. Склейте и
распакуйте на macOS так (распаковывать каждую часть отдельно нельзя):
```bash
# 7-Zip умеет LZMA (его часто используют паки EU4) и сам подхватывает все части,
# если указать на первую:
brew install sevenzip
7zz x "DLC-….zip.001" -o"/путь/к/Europa Universalis IV"   # создаст дерево dlc/

# Либо сначала склейте части в один файл (порядок важен):
cat DLC-….zip.00{1,2,3,4} > DLC-….zip
7zz x DLC-….zip -o"/путь/к/Europa Universalis IV"
```
Указывайте `-o` на папку игры (родитель `dlc/`) — внутри архива уже есть
верхняя папка `dlc/`, поэтому распаковка туда корректно сольёт её на место.

### Почему `status` показывает меньше DLC, чем у меня папок?
Так и должно быть, ничего не потеряно. В папке `dlc/` лежат **паки контента**
(134 для полной игры), но Steam продаёт их как меньшее число **покупок** — одна
покупка часто включает несколько паков (дополнение + его unit-pack + music-pack
имеют один Steam-appid). Разблокировщик выдаёт владение **по Steam-appid**,
поэтому, например, ~75 appid покрывают все 134 пака. `status` показывает число
appid; в лаунчере видны все паки, которые они открывают.

### Steam «Проверить целостность файлов» снёс патч
Это нормально — Steam заменил нашу библиотеку оригиналом. Запустите
`./bin/unlocker status eu4` для проверки и переустановите.

### Игра не запускается
Gatekeeper в macOS может блокировать изменённую библиотеку. Установщик уже
снимает атрибут карантина и ad-hoc переподписывает dylib, но если вы копировали
файлы вручную:
```bash
xattr -dr com.apple.quarantine "/path/to/eu4.app"
codesign --force -s - "/path/to/eu4.app/Contents/Frameworks/libsteam_api.dylib"
```
Если `codesign`/`xattr` отсутствуют — сначала поставьте Command Line Tools:
`xcode-select --install`. Не помогло? Запустите удаление и попробуйте снова.

### Безопасна ли вложенная библиотека?
Поставляемая `libsteam_api.dylib` байт-в-байт совпадает с официальной *nonlog*
сборкой CreamAPI для macOS и даёт **0/64 на VirusTotal**. Контрольную сумму
можно перепроверить самому — см. [docs/SECURITY.md](SECURITY.md) и
[vendor/creamapi/VERSION.txt](../vendor/creamapi/VERSION.txt). (Примечание: в
install-name бинарника зашит путь сборки из апстрима; это безвредно и именно по
нему инструмент определяет установленный патч.)

### Какие DLC разблокируются?
**Все, чей контент есть в папке `dlc/`.** Установщик читает каждый файл
`dlc/*.dlc`, собирает его Steam-appid и пишет весь набор в секцию `[dlc]` файла
`cream_api.ini`. Добавили новый контент DLC? Перезапустите установщик — список
пересоберётся. (`unlockall = true` мы намеренно не используем — рантайм-список
Steam для EU4 обрезан и разблокировал бы лишь часть. `status eu4` показывает,
сколько DLC покрывает текущий конфиг.)

### Можно разблокировать другую игру Paradox?
Движок игронезависимый. Для, например, CK3 нужен лишь новый
`games/<game>.conf` (appid + пути поиска) — без изменений кода. Сегодня
поставляется только EU4; см. [docs/CONTRIBUTING.md](CONTRIBUTING.md).

## 📜 История стратегий разблокировки

У анлокера на macOS сменилось три стратегии разблокировки. Если вы пользовались
старой версией — вот что и почему изменилось:

| Версии    | Стратегия                            | Мультиплеер | Примечания |
|-----------|--------------------------------------|:-----------:|------------|
| 1.x – 2.x | **Goldberg Steam Emulator**          | ❌ | Полностью *эмулирует* Steam офлайн и заменяет `libsteam_api.dylib`; использовал папку `steam_settings/` и `steam_appid.txt`. **Только одиночная игра** — нельзя было приглашать друзей или заходить в сессии. |
| 3.0       | **CreamAPI + `unlockall = true`**    | ✅ | Переход на *прокси* CreamAPI, который передаёт вызовы настоящему Steam — мультиплеер снова работает. Список DLC брался из рантайма Steam. |
| 4.0.1+    | **CreamAPI + явный список `[dlc]`**   | ✅ | Рантайм-список Steam для EU4 обрезан (~⅓ открывалась), поэтому установщик теперь строит полный список из вашего же контента `dlc/`. |

`uninstall` по-прежнему удаляет старые артефакты Goldberg (`steam_settings/`,
`steam_appid.txt`, `*.backup`), если вы обновляетесь с установки 1.x–2.x.

## 🔐 Безопасность и целостность

Установщик не запустится, если SHA256 поставляемой dylib не совпадает с
`vendor/creamapi/VERSION.txt` (защита от подмены/повреждения), а CI
перепроверяет это при каждом пуше. См. [docs/SECURITY.md](SECURITY.md).
